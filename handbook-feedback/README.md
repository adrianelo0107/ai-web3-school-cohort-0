# Handbook Feedback Workflow

目标：把学习过程中遇到的困惑转化为可提交、可复现、可改进的 Handbook feedback。

Goal: turn learning friction into clear, actionable Handbook feedback.

## Flow

1. Capture: 在每日学习时记录具体 Handbook URL、章节标题和困惑点。
2. Triage: 判断属于概念不清、缺少例子、链接问题、结构问题、翻译问题还是前置知识缺口。
3. Draft: 使用 `templates/handbook-feedback.md` 写出观察、学习影响和建议修改。
4. Submit: 手动提交到课程指定渠道、issue、表单或社区讨论区。
5. Track: 把提交链接和后续状态记录到对应 feedback 文件。

## Quality Bar

- Cite the exact section or URL.
- Describe the learner's expectation and the actual friction.
- Propose a concrete edit, example, diagram, or reference.
- Avoid vague feedback such as "this is confusing" without context.
- Keep feedback respectful, specific, and useful for maintainers.

## Naming

Use `YYYY-MM-DD-short-topic.md`, for example:

```text
2026-05-17-agent-workflow-example.md
```
